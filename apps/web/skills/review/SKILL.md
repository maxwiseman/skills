Review the code I've selected or the recent changes for:

- **Bugs & edge cases** — off-by-one errors, null dereferences, unhandled exceptions
- **Security concerns** — injection vectors, exposed secrets, unsafe deserialization
- **Performance issues** — unnecessary re-renders, N+1 queries, memory leaks
- **Readability improvements** — naming clarity, function complexity, dead code

Be concise and actionable. Group findings by severity (critical, warning, suggestion). Skip praise — only flag real issues.
